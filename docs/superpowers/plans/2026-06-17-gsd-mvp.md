# GSD MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a repo-local `gsd` CLI that initializes delivery folders, starts a change, runs verification checks, records evidence, and writes done reports.

**Architecture:** Use a small Node.js CLI with no external dependencies. Core behavior lives in `src/gsd.mjs` as testable functions, and `bin/gsd.mjs` only parses CLI arguments and prints results.

**Tech Stack:** Node.js ESM, `node:test`, built-in `fs`, `path`, `child_process`, and JSON/YAML-like text files.

## Global Constraints

- No network dependency is required for the MVP.
- All generated project state is repo-local under `.agent/`, `.gsd/`, and `openspec/`.
- Verification must write evidence before `done` can complete.
- Default checks are configured in `.gsd/workflow.json`.
- Full E2E-style checks are supported through `gsd verify --full`.
